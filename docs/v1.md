# PetVerse — 全域宠物管理平台 项目实施 Prompt

> 本文档为交给 Codex / Claude Code 的完整项目 Prompt。
> 后端使用 Golang，前端使用 Expo (React Native)，采用 monorepo 结构。

---

## 1. 项目总览

### 1.1 产品定义

PetVerse 是一款一站式 AI 驱动的宠物管理超级应用，整合健康管理、智能硬件控制、服务预约、训练指导、社区互动和智能购物六大核心模块。

### 1.2 技术栈

| 层级 | 技术选型 |
|------|----------|
| 前端 | Expo SDK 52+, React Native, TypeScript, Expo Router (file-based routing) |
| 状态管理 | Zustand + React Query (TanStack Query) |
| 前端 UI | Tamagui 或 NativeWind (Tailwind CSS for RN) |
| 后端 | Go 1.22+, Gin (HTTP framework), GORM (ORM) |
| 数据库 | PostgreSQL 16 (主库), Redis 7 (缓存/会话/实时), TimescaleDB (IoT 时序数据) |
| 消息队列 | NATS / Redis Streams |
| 对象存储 | MinIO / S3 兼容 (图片、视频、文档) |
| 认证 | JWT + Refresh Token，支持手机号/邮箱/微信/Apple/Google 登录 |
| API 协议 | RESTful JSON API，WebSocket (实时推送) |
| 部署 | Docker Compose (开发), Kubernetes (生产) |
| CI/CD | GitHub Actions |

## 2. 后端详细设计 (Golang)

### 2.1 数据模型

请按照以下 schema 实现所有数据模型。使用 GORM 标签，支持软删除。

```go
// ==================== 用户体系 ====================

type User struct {
    ID          uuid.UUID      `gorm:"type:uuid;primaryKey;default:gen_random_uuid()"`
    Phone       *string        `gorm:"uniqueIndex;size:20"`
    Email       *string        `gorm:"uniqueIndex;size:255"`
    Password    string         `gorm:"size:255"`          // bcrypt hashed
    Nickname    string         `gorm:"size:50"`
    AvatarURL   string         `gorm:"size:500"`
    Role        string         `gorm:"size:20;default:'user'"` // user, merchant, admin
    WechatOpenID *string       `gorm:"uniqueIndex;size:100"`
    AppleID     *string        `gorm:"uniqueIndex;size:100"`
    GoogleID    *string        `gorm:"uniqueIndex;size:100"`
    Latitude    *float64
    Longitude   *float64
    PlanType    string         `gorm:"size:20;default:'free'"` // free, pro, merchant
    PlanExpiry  *time.Time
    CreatedAt   time.Time
    UpdatedAt   time.Time
    DeletedAt   gorm.DeletedAt `gorm:"index"`
    Pets        []Pet          `gorm:"foreignKey:OwnerID"`
}

// ==================== 宠物管理 ====================

type Pet struct {
    ID          uuid.UUID      `gorm:"type:uuid;primaryKey;default:gen_random_uuid()"`
    OwnerID     uuid.UUID      `gorm:"type:uuid;index;not null"`
    Name        string         `gorm:"size:50;not null"`
    Species     string         `gorm:"size:20;not null"`   // dog, cat, bird, rabbit, etc.
    Breed       string         `gorm:"size:50"`
    Gender      string         `gorm:"size:10"`            // male, female, unknown
    BirthDate   *time.Time
    Weight      *float64                                   // kg
    AvatarURL   string         `gorm:"size:500"`
    Microchip   *string        `gorm:"size:50"`
    IsNeutered  bool           `gorm:"default:false"`
    Allergies   string         `gorm:"type:text"`          // JSON array
    Notes       string         `gorm:"type:text"`
    HealthScore *int                                       // 0-100, AI 计算
    CreatedAt   time.Time
    UpdatedAt   time.Time
    DeletedAt   gorm.DeletedAt `gorm:"index"`
    Owner       User           `gorm:"foreignKey:OwnerID"`
}

// ==================== 健康档案 ====================

type HealthRecord struct {
    ID          uuid.UUID      `gorm:"type:uuid;primaryKey;default:gen_random_uuid()"`
    PetID       uuid.UUID      `gorm:"type:uuid;index;not null"`
    Type        string         `gorm:"size:30;not null"`
    // Types: vaccination, medication, checkup, surgery, weight, symptom, diet, lab_result
    Title       string         `gorm:"size:200;not null"`
    Description string         `gorm:"type:text"`
    Data        datatypes.JSON                             // 灵活的 JSON 数据字段
    DueDate     *time.Time                                 // 下次提醒日期
    ProviderID  *uuid.UUID     `gorm:"type:uuid"`          // 关联服务商
    Attachments datatypes.JSON                             // 图片/文件 URL 数组
    RecordedAt  time.Time      `gorm:"not null"`
    CreatedAt   time.Time
    UpdatedAt   time.Time
    DeletedAt   gorm.DeletedAt `gorm:"index"`
    Pet         Pet            `gorm:"foreignKey:PetID"`
}

type HealthAlert struct {
    ID          uuid.UUID      `gorm:"type:uuid;primaryKey;default:gen_random_uuid()"`
    PetID       uuid.UUID      `gorm:"type:uuid;index;not null"`
    AlertType   string         `gorm:"size:30;not null"`   // ai_warning, reminder, anomaly
    Severity    string         `gorm:"size:10;not null"`   // low, medium, high, critical
    Title       string         `gorm:"size:200;not null"`
    Message     string         `gorm:"type:text;not null"`
    Source      string         `gorm:"size:30"`            // ai, device, manual
    IsRead      bool           `gorm:"default:false"`
    IsDismissed bool           `gorm:"default:false"`
    CreatedAt   time.Time
    Pet         Pet            `gorm:"foreignKey:PetID"`
}

// ==================== 智能设备 ====================

type Device struct {
    ID           uuid.UUID      `gorm:"type:uuid;primaryKey;default:gen_random_uuid()"`
    OwnerID      uuid.UUID      `gorm:"type:uuid;index;not null"`
    PetID        *uuid.UUID     `gorm:"type:uuid;index"`     // 关联宠物（可选）
    DeviceType   string         `gorm:"size:30;not null"`
    // Types: feeder, water_fountain, litter_box, camera, gps_collar, door
    Brand        string         `gorm:"size:50"`
    Model        string         `gorm:"size:50"`
    Nickname     string         `gorm:"size:50"`
    SerialNumber string         `gorm:"size:100;uniqueIndex"`
    Status       string         `gorm:"size:20;default:'offline'"` // online, offline, error
    Config       datatypes.JSON                              // 设备配置 JSON
    LastSeen     *time.Time
    FirmwareVer  string         `gorm:"size:20"`
    BatteryLevel *int                                        // 0-100, null if wired
    CreatedAt    time.Time
    UpdatedAt    time.Time
    DeletedAt    gorm.DeletedAt `gorm:"index"`
    Owner        User           `gorm:"foreignKey:OwnerID"`
    Pet          *Pet           `gorm:"foreignKey:PetID"`
}

// TimescaleDB 超表 — IoT 数据点
type DeviceDataPoint struct {
    Time     time.Time `gorm:"not null;index:idx_device_data_time,priority:2"`
    DeviceID uuid.UUID `gorm:"type:uuid;not null;index:idx_device_data_time,priority:1"`
    Metric   string    `gorm:"size:50;not null"` // feeding_amount, water_intake, litter_usage, motion, location_lat, location_lng, etc.
    Value    float64   `gorm:"not null"`
    Unit     string    `gorm:"size:20"`
    Meta     datatypes.JSON
}

// ==================== 服务市场 ====================

type ServiceProvider struct {
    ID          uuid.UUID      `gorm:"type:uuid;primaryKey;default:gen_random_uuid()"`
    UserID      uuid.UUID      `gorm:"type:uuid;uniqueIndex;not null"` // 关联商家用户
    Name        string         `gorm:"size:100;not null"`
    Type        string         `gorm:"size:30;not null"`
    // Types: vet_clinic, grooming, boarding, walking, training, pet_store
    Description string         `gorm:"type:text"`
    Address     string         `gorm:"size:300"`
    Latitude    float64
    Longitude   float64
    Phone       string         `gorm:"size:20"`
    Photos      datatypes.JSON                             // URL 数组
    Rating      float64        `gorm:"default:0"`
    ReviewCount int            `gorm:"default:0"`
    IsVerified  bool           `gorm:"default:false"`
    OpenHours   datatypes.JSON                             // 营业时间 JSON
    Services    datatypes.JSON                             // 服务项目与价格
    Tags        datatypes.JSON                             // 标签数组
    CreatedAt   time.Time
    UpdatedAt   time.Time
    DeletedAt   gorm.DeletedAt `gorm:"index"`
    User        User           `gorm:"foreignKey:UserID"`
}

type Booking struct {
    ID             uuid.UUID      `gorm:"type:uuid;primaryKey;default:gen_random_uuid()"`
    UserID         uuid.UUID      `gorm:"type:uuid;index;not null"`
    PetID          uuid.UUID      `gorm:"type:uuid;index;not null"`
    ProviderID     uuid.UUID      `gorm:"type:uuid;index;not null"`
    ServiceName    string         `gorm:"size:100;not null"`
    Status         string         `gorm:"size:20;default:'pending'"`
    // Status: pending, confirmed, in_progress, completed, cancelled, no_show
    StartTime      time.Time      `gorm:"not null"`
    EndTime        *time.Time
    Price          int64                                    // 分为单位
    Currency       string         `gorm:"size:3;default:'CNY'"`
    Notes          string         `gorm:"type:text"`
    CancelReason   string         `gorm:"type:text"`
    Rating         *int                                    // 1-5
    Review         string         `gorm:"type:text"`
    CreatedAt      time.Time
    UpdatedAt      time.Time
    User           User            `gorm:"foreignKey:UserID"`
    Pet            Pet             `gorm:"foreignKey:PetID"`
    Provider       ServiceProvider `gorm:"foreignKey:ProviderID"`
}

// ==================== 训练 ====================

type TrainingPlan struct {
    ID          uuid.UUID      `gorm:"type:uuid;primaryKey;default:gen_random_uuid()"`
    PetID       uuid.UUID      `gorm:"type:uuid;index;not null"`
    Title       string         `gorm:"size:100;not null"`
    Description string         `gorm:"type:text"`
    Difficulty  string         `gorm:"size:20"`           // beginner, intermediate, advanced
    Category    string         `gorm:"size:30"`           // obedience, tricks, behavior, agility
    Steps       datatypes.JSON                            // 训练步骤 JSON 数组
    AIGenerated bool           `gorm:"default:false"`
    Progress    int            `gorm:"default:0"`         // 0-100
    CreatedAt   time.Time
    UpdatedAt   time.Time
    Pet         Pet            `gorm:"foreignKey:PetID"`
}

// ==================== 社区 ====================

type Post struct {
    ID          uuid.UUID      `gorm:"type:uuid;primaryKey;default:gen_random_uuid()"`
    AuthorID    uuid.UUID      `gorm:"type:uuid;index;not null"`
    PetID       *uuid.UUID     `gorm:"type:uuid;index"`
    Type        string         `gorm:"size:20;default:'post'"` // post, question, guide, event
    Title       string         `gorm:"size:200"`
    Content     string         `gorm:"type:text;not null"`
    Images      datatypes.JSON                             // URL 数组
    Tags        datatypes.JSON
    Latitude    *float64
    Longitude   *float64
    LikeCount   int            `gorm:"default:0"`
    CommentCount int           `gorm:"default:0"`
    IsPublished bool           `gorm:"default:true"`
    CreatedAt   time.Time

    DeletedAt   gorm.DeletedAt `gorm:"index"`

type Comment struct {
    ID        uuid.UUID      `gorm:"type:uuid;primaryKey;default:gen_random_uuid()"`
    PostID    uuid.UUID      `gorm:"type:uuid;index;not null"`
    AuthorID  uuid.UUID      `gorm:"type:uuid;index;not null"`
    ParentID  *uuid.UUID     `gorm:"type:uuid;index"`       // 支持嵌套回复
    Content   string         `gorm:"type:text;not null"`
    LikeCount int            `gorm:"default:0"`
    CreatedAt time.Time
    DeletedAt gorm.DeletedAt `gorm:"index"`

type Like struct {
    UserID    uuid.UUID `gorm:"type:uuid;primaryKey"`

// ==================== 商城 ====================

type Product struct {
    ID          uuid.UUID      `gorm:"type:uuid;primaryKey;default:gen_random_uuid()"`

    Name        string         `gorm:"size:200;not null"`
    Description string         `gorm:"type:text"`

    // Categories: food, treats, toys, grooming, health, accessories

    // Types: health_alert, booking_update, community, system, promotion

    IsRead    bool           `gorm:"default:false"`
```

### 2.3 后端实现要求

1. **项目结构**：严格遵循 `cmd/internal` 的 Go 标准项目布局。handler 层不含业务逻辑，只做参数校验和响应包装；service 层处理所有业务逻辑；repository 层负责数据库操作。

2. **依赖注入**：使用构造函数注入，不使用全局变量。在 `main.go` 中组装所有依赖。

3. **统一响应格式**：
```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "meta": {
    "page": 1,
    "page_size": 20,
    "total": 100
  }
}
```

4. **错误处理**：定义统一的 AppError 类型，包含 HTTP 状态码、业务错误码和用户可读消息。使用 Gin 的 recovery 中间件捕获 panic。

5. **中间件**：
   - `AuthMiddleware`：JWT 验证，从 Authorization header 提取 token
   - `CORSMiddleware`：跨域处理
   - `RateLimitMiddleware`：基于 Redis 的滑动窗口限流
   - `LoggerMiddleware`：结构化日志 (使用 `slog`)
   - `RequestIDMiddleware`：为每个请求生成唯一 ID

6. **配置管理**：使用 `viper` 读取 YAML 配置文件和环境变量。支持 dev/staging/prod 多环境。

7. **数据库迁移**：使用 `golang-migrate` 管理 SQL 迁移文件，放在 `migrations/` 目录。

8. **AI 服务封装**：在 `internal/pkg/ai/` 中封装对 Claude API 的调用，支持：
   - 健康分析：传入宠物的健康记录和设备数据，返回健康洞察
   - 训练计划生成：根据宠物品种、年龄、性格生成训练方案
   - 知识问答：基于宠物养护知识回答用户问题
   - 商品推荐：基于宠物档案推荐合适商品

9. **WebSocket**：使用 `gorilla/websocket`，实现 Hub 模式管理连接。支持按用户 ID 推送通知，按设备 ID 推送实时数据。

10. **单元测试**：每个 service 层方法都要有对应的单元测试，使用 `testify` 和 `mockery` 生成 mock。

## 5. 编码规范

### Go 后端

- 遵循 Effective Go 和 Go Code Review Comments
- 使用 `golangci-lint` 检查代码质量
- 错误必须处理，不允许 `_ = err`
- Context 传递：所有 service/repo 方法第一个参数必须是 `ctx context.Context`
- 日志使用 `slog`，结构化输出
- 变量/函数命名使用 camelCase，导出使用 PascalCase
- 每个包的文件按功能命名，不按类型 (不要 `models.go` 放所有模型)

### TypeScript 前端

- 严格模式 (`strict: true`)
- 使用函数组件 + Hooks，不使用 class 组件
- 文件命名 kebab-case，组件导出 PascalCase
- 路由文件按 Expo Router 约定放在 `app/` 目录
- Props 接口命名为 `XxxProps`
- 使用绝对路径导入 (`@/components/...`)
- 所有异步操作使用 React Query，不在组件内使用裸 `useEffect` + `fetch`
